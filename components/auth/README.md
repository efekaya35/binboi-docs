# `components/auth`

Purpose: reusable auth presentation components and shared form shells.

What belongs here:
- Form wrappers, auth cards, helper panels, and any auth-only client components.
- Shared UX for field errors, action state messaging, and auth layout polish.

Notes:
- `auth-form.tsx` is the primary reusable primitive for auth pages.
- Keep validation messaging crisp and product-friendly instead of generic template copy.
